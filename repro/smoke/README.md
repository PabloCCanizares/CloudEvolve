# Simulator smoke fixture (ready to run)

A self-contained, **pre-decompressed** setup that runs the real CloudSim-Storage
simulator on the `Al_w3` initial-population case instantly — no unzip, no path
rewriting.

```
repro/smoke/
├── run.sh                         # launch the simulator on Al_w3 (instant)
├── Al_w3/
│   ├── input.tc                   # real Al_w3 config, relative work.path
│   ├── meta.mtc                   # relative InputSrc/OutputSrc
│   └── output.tc                  # generated by a run (git-ignored)
└── workload/io_mix/cpu/mix_vm/    # 20 workload traces the case needs
```

All data paths inside `input.tc` / `meta.mtc` are **relative to this folder**, so
the simulator must be launched with `repro/smoke` as the working directory
(`run.sh` does `cd` for you).

## Run it

```bash
./repro/smoke/run.sh
# -> total Energy consumption (CPU+storage): ~23.89 kWh
# -> Total simulation time: ~2770 s
```

Or from the JUnit suite (same fixture, asserts the figures are sane):

```bash
mvn test -Dce.runSimulatorTests=true -Dtest=SimulatorIntegrationSmokeTest
```

## Notes

* **Locale** — the simulator prints decimals with the default locale and its
  own parser expects a dot, so the run is pinned to `en_US`.
* **Heap** — the full-size case (2048 VMs / 512 hosts) needs `-Xmx2g`.
* The traces are a 20-file subset of `repro/workload.zip` (the simulator samples
  `work.numTraces` of them); enough to run reproducibly while staying small.
